## ✨ Features

| Feature | Description |
|---|---|
| 🤖 **AI Resume Tailoring** | Paste a job description → get a custom-tailored CV generated by Cloudflare Workers AI (LLaMA 3.2) |
| 📄 **PDF Upload** | Client-side PDF text extraction via `pdfjs-dist` — no file ever leaves your browser |
| 💬 **AI Resume Chat** | Ask questions about your resume and receive expert feedback |
| 🔐 **Secure Auth** | Email + password registration & login with PBKDF2 hashing, JWT sessions, and rate limiting |
| 🕒 **Resume History** | All generated resumes saved to Cloudflare D1, accessible any time |
| 🖨️ **Print-Ready A4** | Download or print a pixel-perfect A4 resume directly from the browser |
| 🌙 **Dark Glassmorphism UI** | Premium dark mode UI with micro-animations and gradient accents |

## 🔐 Security

| Layer | Mechanism |
|---|---|
| **Password hashing** | PBKDF2-SHA256, 100 000 iterations, random 16-byte salt (Web Crypto API) |
| **Auth tokens** | HS256 JWT, 24 h expiry, signed with `JWT_SECRET` env var |
| **Brute-force protection** | 5 auth attempts / IP / 15 min — enforced via Cloudflare KV |
| **Email enumeration** | Generic error messages on both register and login |
| **SQL injection** | Cloudflare D1 prepared statements with bound parameters |
| **CORS** | Locked to `FRONTEND_URL` in production; credentials-aware |
| **Input size caps** | 50 KB max per AI text payload; 200 char max on save names |
| **Rate limiting (AI)** | 10 AI generations + 15 resume queries per user per day |
| **Client-side files** | PDF processing is 100% in-browser — no file upload to server |

### Frontend
- **React 19** with TypeScript
- **Vite 8** for bundling
- **Vanilla CSS** with CSS custom properties and glassmorphism
- **pdfjs-dist** for client-side PDF parsing
- **lucide-react** for icons
- **Google Fonts** — Inter + Outfit

### Backend
- **Hono.js 4** on **Cloudflare Workers**
- **Cloudflare D1** (SQLite) for user data & resume history
- **Cloudflare KV** for rate limiting
- **Cloudflare Workers AI** — `@cf/meta/llama-3.2-3b-instruct`
- **Web Crypto API** for PBKDF2 hashing & HS256 JWT signing (zero dependencies)

## 🚀 Quick Start (Local Development)

### Prerequisites
- Node.js 18+
- [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/) v3+ (`npm i -g wrangler`)

### 1. Clone & install

```bash
git clone https://github.com/ChandrashekharaKM/ResumeBuilder.git
cd ResumeBuilder

# Install backend deps
cd backend && npm install

# Install frontend deps
cd ../frontend && npm install
```

### 2. Initialise the local database

```bash
cd backend

# Apply the schema and all migrations to the local D1 instance
wrangler d1 execute holdmyresume-db --local --file=schema.sql
wrangler d1 migrations apply holdmyresume-db --local
```

### 3. Configure frontend environment

```bash
cd ../frontend
cp .env.example .env
# VITE_API_URL is already set to http://localhost:8787 in .env.example
```

### 4. Run both services

```bash
# Terminal 1 — backend (http://localhost:8787)
cd backend && npm run dev

# Terminal 2 — frontend (http://localhost:5173)
cd frontend && npm run dev
```

Open **http://localhost:5173**, create an account, and start building resumes.

## 🌐 Production Deployment

See **[DEPLOYMENT.md](./DEPLOYMENT.md)** for the full step-by-step guide covering:

- Cloudflare D1 + KV + Workers AI setup
- `wrangler secret put JWT_SECRET` for production secrets
- Cloudflare Pages deployment (or GitHub auto-deploy)
- Security checklist before go-live

**Quick summary:**
```bash
# Backend
cd backend
wrangler d1 create holdmyresume-db
wrangler d1 execute holdmyresume-db --file=schema.sql --remote
wrangler d1 migrations apply holdmyresume-db --remote
wrangler secret put JWT_SECRET
wrangler secret put FRONTEND_URL
wrangler deploy

# Frontend
cd frontend
npm run build
wrangler pages deploy dist --project-name holdmyresume
```

## 🔑 Environment Variables

### Backend (`wrangler.toml` / Wrangler secrets)

| Variable | Where | Description |
|---|---|---|
| `JWT_SECRET` | Wrangler secret | HS256 signing key — use `openssl rand -hex 32` |
| `FRONTEND_URL` | Wrangler secret | Your Pages URL for CORS allowlist |
| `DB` | wrangler.toml binding | Cloudflare D1 database |
| `LIMITER_KV` | wrangler.toml binding | Cloudflare KV for rate limiting |
| `AI` | wrangler.toml binding | Cloudflare Workers AI |

### Frontend (`.env`)

| Variable | Default | Description |
|---|---|---|
| `VITE_API_URL` | `http://localhost:8787` | Backend Worker URL |

> ⚠️ **Never commit `.env` or `.dev.vars`** — they are excluded by `.gitignore`.
